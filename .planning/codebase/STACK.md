# Technology Stack

**Analysis Date:** 2026-04-27

## Languages

**Primary:**
- C# - Primary scripting language for all agent simulation logic, managers, and analytics
- GLSL - Shader language (implicitly, via Universal Render Pipeline)

**Supporting:**
- JSON - Configuration files and output serialization (via `JsonUtility`)
- CSV - Data ingestion for sensor readings and session schedules (custom parsing)
- GLTF - 3D venue geometry (Hall models imported as assets)

## Runtime

**Environment:**
- Unity Engine 6.0.3.10f1 (LTS - verified in `ProjectSettings/ProjectVersion.txt`)
  - Supports .NET Framework and .NET Standard 2.1
  - Uses MonoBehaviour component system
  
**Development Tools:**
- Visual Studio (via `.vsconfig` - Workload.ManagedGame)
- Rider IDE 3.0.39 (via `com.unity.ide.rider` package)
- Visual Studio Code (via `com.unity.ide.visualstudio` 2.0.26)

**Graphics & Rendering:**

- ProBuilder 6.0.9 - Scene geometry generation and manipulation

**Development & Tooling:**

**Testing:**

**Input & UI:**

**Optional/Experimental:**

## Key Dependencies

**Critical:**
- `com.unity.ai.navigation` (2.0.10) - Pathfinding backbone. All agent movement depends on NavMesh baking via `Editor/NavMeshBaker.cs`. **Without this, agents cannot navigate.**
- `com.unity.render-pipelines.universal` (17.3.0) - Rendering pipeline. Required for real-time venue visualization and heatmap texture rendering.

**Analytics & Data:**
- No external CSV/JSON libraries - all parsing is custom-coded via `System.IO`, `System.Globalization`, `System.Text` in:
  - `DataLoader.cs` - Parses `sensor_data.csv` (headcount timeseries)
  - `SessionScheduleLoader.cs` - Parses `session_schedule.csv` (conference schedule)
  - `AnalyticsManager.cs` - Exports CSV files and heatmap PNG via `Texture2D.EncodeToPNG()`

**Physics & Movement:**
- `UnityEngine.AI` namespace - NavMesh agent queries (`NavMesh.FindClosestEdge`, `NavMeshAgent`)
- `UnityEngine.Physics` - Raycast and collision detection for social force model

**Serialization:**
- `UnityEngine.JsonUtility` - Built-in JSON serialization for `SimConfig` and output reports
  - No Newtonsoft.Json or System.Text.Json dependencies
  - Manually parsed JSON in `LLMConversationClient.cs` for API responses (avoids JSON library)

**Module System:**
- Native C# (`System` namespace) - No game-specific module system
- Assembly-CSharp and Assembly-CSharp-Editor compiled at runtime by Unity

## Configuration

**Environment:**
- `SimConfig` (ScriptableObject) - Central parameter store with 50+ tunable fields:
  - Social Force Model parameters (Helbing et al. - desiredSpeed, socialForceA, socialForceB, etc.)
  - Fatigue model (fatigueTimeConstant, minSpeedFraction)
  - Dwell time log-normal distribution (dwellMu=5.70, dwellSigma=0.80)
  - Roaming behavior (Ornstein-Uhlenbeck process)
  - Density thresholds and crowding sigmoid
  - Persona configurations (5 types: Researcher, Networker, Student, Industry, BoothStaff)
  - Day-phase rhythm (Opening, Morning, Lunch, Afternoon, Evening)
  - Physiological drives (hunger, thirst rates and thresholds)
  - Social group cohesion and separation parameters
  - LLM configuration (API key, model, call budget)

**Runtime Overrides:**
- `Assets/StreamingAssets/sim_config.json` - Optional JSON overrides at startup (parsed by `ConfigLoader.cs`)
  - Example: `agentSpeed`, `personalBubble`, `fatiguePerSecond`, `minSpeedMultiplier`

**Build:**
- `Assembly-CSharp.csproj` - Auto-generated by Unity for game scripts
- `Assembly-CSharp-Editor.csproj` - Editor-only scripts (setup tools, nav mesh baker)
- Both projects reference Unity assemblies and core .NET Framework
